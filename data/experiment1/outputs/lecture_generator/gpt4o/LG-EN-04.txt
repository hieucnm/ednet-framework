## Lecture: Field-Programmable Gate Arrays: Architecture and Programming Model
**Course:** Semiconductor Design Fundamentals — VASTN Joint Certificate  
**Audience level:** Undergraduate  
**Duration:** 60 minutes  

### Learning Objectives
By the end of this lecture, learners will be able to describe the internal structure of an FPGA, including logic blocks, routing fabric, and I/O cells. They will also be able to explain the differences between FPGA-based and ASIC-based implementations, identify when each is appropriate, write a simple combinational logic function in VHDL, simulate its behavior, and interpret a basic timing report from an FPGA synthesis tool.

### Part 1 — Introduction to FPGA Architecture (~20 minutes)
This section aims to provide a foundational understanding of the internal structure of FPGAs. Field-Programmable Gate Arrays (FPGAs) are integrated circuits that can be configured by the customer or designer after manufacturing. The primary components of an FPGA include logic blocks, routing fabric, and I/O cells. Logic blocks are the basic units of computation and can implement simple logic functions. The routing fabric connects these logic blocks, allowing for complex circuit designs. I/O cells manage the interface between the FPGA and external components. Understanding these components is crucial for designing and implementing digital circuits on FPGAs.

### Part 2 — FPGA vs. ASIC: When to Use Each (~20 minutes)
The purpose of this section is to differentiate between FPGA-based and ASIC-based implementations and discuss their appropriate applications. FPGAs offer flexibility and reconfigurability, making them ideal for prototyping and applications requiring frequent updates. In contrast, Application-Specific Integrated Circuits (ASICs) are custom-designed for specific tasks, offering higher performance and lower power consumption but at the cost of flexibility. FPGAs are suitable for low to medium volume production, while ASICs are more cost-effective for high-volume production due to their higher initial design costs. Understanding these trade-offs helps in making informed decisions about which technology to use for a given application.

### Part 3 — Programming FPGAs with VHDL (~20 minutes)
This section focuses on the practical aspect of programming FPGAs using VHDL (VHSIC Hardware Description Language). VHDL is a hardware description language used to model electronic systems. Learners will write a simple combinational logic function in VHDL and simulate its behavior using an FPGA synthesis tool. The process involves writing the VHDL code, compiling it, and then simulating the logic to verify its correctness. Additionally, learners will learn how to interpret a basic timing report generated by the synthesis tool, which provides insights into the performance of the implemented design. This hands-on experience is essential for understanding how theoretical concepts translate into practical applications.
